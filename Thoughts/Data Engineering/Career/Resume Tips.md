# Resume & Portfolio Tips

> *Stand out in the job market*

---

## Resume Structure

### Header
- Name, Email, Phone, Location
- LinkedIn, GitHub links
- Portfolio/Blog (if any)

### Summary (Optional)
- 2-3 lines max
- Years of experience + top skills
- Notable achievement

### Skills Section

```
Languages: Python, SQL, Scala
Databases: PostgreSQL, Snowflake, BigQuery
Processing: Apache Spark, Apache Flink
Orchestration: Apache Airflow, Prefect
Cloud: AWS (S3, Glue, EMR, Redshift), GCP (BigQuery)
Tools: Docker, Kubernetes, Terraform, dbt, Git
```

### Experience (STAR Format)

**Data Engineer** | Company | Dates

- **Built** ETL pipeline processing 10M records/day using Spark and Airflow, reducing processing time by 60%
- **Designed** dimensional model for customer analytics, enabling self-serve reporting for 50+ analysts
- **Implemented** data quality framework with Great Expectations, achieving 99.9% data accuracy
- **Migrated** legacy Oracle warehouse to Snowflake, saving $100K/year in licensing

### Projects

For each project:
- **What**: One-line description
- **Tech**: Technologies used
- **Impact**: Measurable outcome

## LinkedIn Optimization

1. **Headline**: Data Engineer | Python, Spark, Airflow | Building Data Pipelines
2. **About**: 3-4 paragraphs with keywords
3. **Experience**: Similar to resume
4. **Skills**: Add all relevant, get endorsements
5. **Projects**: Add as featured content

---

## Common Mistakes

❌ Listing technologies without context  
✅ Showing impact with technologies

❌ "Worked on data pipelines"  
✅ "Built Spark pipeline processing 1TB daily"

❌ Generic resume for all jobs  
✅ Tailored keywords from job description

❌ No metrics or outcomes  
✅ Quantified achievements (%, $, time)
